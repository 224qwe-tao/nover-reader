# 本機小說閱讀器

這是一個可以放到 GitHub Pages 的純前端小說閱讀網站。使用者手動選擇電腦中的小說檔案後，網站會在瀏覽器內讀取內容，不會把小說上傳到任何伺服器。

## 功能

- 支援 TXT、DOCX、EPUB
- TXT 可選 UTF-8、Big5、GB18030 編碼
- 自動偵測章節，例如：第1章、第十二章、第三百零五章、Chapter 1
- 章節顯示在「章節列表」
- 可搜尋章節
- 上一章 / 下一章
- 複製目前章節
- 字體大小和行距調整
- 自動填滿內容寬度：會把 TXT 裡固定換行造成的右側空白整理成自動換行
- 左側欄改為類似 PDF 閱讀器的展開 / 收起效果：收起後只保留一條窄工具欄和選單按鈕
- 載入小說、保存 / 書庫、閱讀設定、章節列表四個方格可分別收起 / 打開
- 全螢幕閱讀模式：進入後會隱藏網站標題、頁尾和左側欄，只保留閱讀區
- 深色模式
- 本機書庫保存功能：可保存已載入小說、命名、分類，下次開同一個網站時可直接從書庫打開
- 已保存小說可刪除、改名、改分類

## 使用方法

1. 開啟 `index.html`。
2. 按「選擇小說檔案」。
3. 選擇 TXT、DOCX 或 EPUB。
4. 網站會自動解析章節，並在左側章節列表顯示。
5. 在「保存 / 書庫」輸入保存名稱和分類。
6. 按「保存」，小說會保存到目前瀏覽器的本機書庫。
7. 下次打開同一個網站時，可在左側「保存 / 書庫」直接按「打開」。

## 部署到 GitHub Pages

1. 建立一個 GitHub repository。
2. 上傳 `index.html`、`style.css`、`app.js`、`README.md`。
3. 到 repository 的 `Settings` → `Pages`。
4. Source 選擇要發佈的 branch，例如 `main`。
5. 儲存後等待 GitHub 生成網站網址。

## 本機保存說明

- 保存功能使用瀏覽器 IndexedDB。
- 小說內容只會保存在使用者目前的瀏覽器和裝置內，不會上傳到 GitHub。
- 更換瀏覽器、使用無痕模式、清除網站資料或清除瀏覽器資料後，保存的小說可能會消失。
- 如果想長期備份小說，仍建議保留原本的 TXT、DOCX 或 EPUB 檔案。

## 注意

- `.docx` 需要透過 CDN 載入 Mammoth.js。
- `.epub` 需要透過 CDN 載入 JSZip。
- 舊式 `.doc` 是二進制 Word 格式，不建議在純前端網頁解析；請先另存為 `.docx` 或 `.txt`。
